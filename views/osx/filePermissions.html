OS X File Permissions
June 10, 2016

Quick Version

  • chmod 741 sometext.txt
    • The first column sets permissions for owner |7|41
    • The second column sets permissions for group 7|4|1
    • The third column sets permissions for other 74|1|
    • The read, write, and execute permissions are constructed from the sum of the following values:
      • 4 allows read rights
      • 2 allows write rights
      • 1 allows execute rights
    • The sum of all three values is 7 indicating permission to read, write, and execute the file.
  • chmod 400 sometext.txt would set the permissions of the owner to read only and not give any permissions to group or other.

Detailed Version

Permissions Terminology
  1. Open the terminal
  2. Type cd /
  3. Type ls -l
  4. You should get a list of files
  5. Let's break apart the first column. This is called the file mode and should look something like: drwxr-xr-x@
  • d - this part is called the entry type. This character describes the type of file:
    • b block special file
    • c character special file
    • d directory
    • l symbolic link
    • s socket link
    • p FIFO
    • - regular file
  • rwxr-xr-x - permissions
    • rwx - owner permissions
    • r-x - group permissions
    • r-x - other permissions
    • Each of the permission fields have three character positions:
      • r - the file is readable, if not it is denoted as -
      • w - the file is writable, if not it is denoted as -
      • x - this position can take on different characters:
        • S If in the owner permissions, the file is not executable and set-user-ID mode is set. If in the group permissions, the file is not executable and set-group-ID mode is set.
        • s If in the owner permissions, the file is executable and set-user-ID mode is set. If in the group permissions, the file is executable and setgroup-ID mode is set.
        • x The file is executable or the directory is searchable.
        • - The file is neither readable, writable, executable, nor set-user-ID nor set-group-ID mode, nor sticky.
  • @ - this field could potentially be empty
    • If the field is empty, the file uses the standard set of owner/group/other
    • @ - indicates the file has extended attributes
    • + - indicates the richer permission model (access control lists)
More details regarding the long format can be found by opening the terminal and typing: man ls

Setting the permissions
  • cd ~
  • vim test.txt
    • :wq then press enter to save the file
  • chmod 777 test.txt would set read, write, and execute permissions to owner, group, and other
    • The first column in |7|77 sets permissions for owner
    • The second column 7|7|7 sets permissions for group
    • The third column 77|7| sets permissions for other
    • The read, write, and execute permissions are constructed from the sum of the following values:
      • 4 allows read rights
      • 2 allows write rights
      • 1 allows execute rights
    • The sum of all three values is 7 indicating permission to read, write, and execute the file. If you want read only then the sum would be 4.
  • Let's make the file read only for owner and give no permissions to group or other.
    • chmod 400 test.txt
    • ls -l
      • the file mode should look something like -r--------
    • Go ahead and try editing the file: vim test.txt
    • You'll notice that the file isn't writable, that's because we set permissions to read only!
  • It's worth noting you can set user-id and group-id permissions, to find more information regarding that: man chmod
