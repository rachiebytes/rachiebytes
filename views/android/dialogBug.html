Android Studio Dialog Bug
May 29th, 2016

FYI: I'm using v1.5 and a Mac

If Android Studio is in full screen mode and you have a dialog box open (e.g. the Generate Signed APK dialog) then go to another screen (by swiping or CMD + tab), then go back to the IDE you'll notice it seems to be locked up.

The dialog box is hidden and you'll pretty much be unable to use the IDE. I found that if I use CMD + W. It'll close the dialog box and unlock the IDE (however this will lose any progress made in the dialog box). This is a known issue.
